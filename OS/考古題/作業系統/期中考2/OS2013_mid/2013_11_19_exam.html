問題 解答
1a. 何謂 multi-programming, multi-processing? p. 19,  OS放多個processes於主記憶體上, 以方便切換CPU給各process使用. 
p. 17,  系統裡有多個一般目的之processors, 可用以同時執行多個processes.
1b. 製作time-sharing OS 需要什麼電路支援? 為什麼?  可定時發出中斷之 timer電路.   計時終了 timer 就發出中斷, 可避免CPU被永遠佔據
1c. 會讓CPU離開user mode 而進入kernel mode 的事件或動作. 硬體中斷, 系統呼叫, 非法之記憶體存取, 算數除0錯誤,
 
2a. 為何要使用Win32 API 而不要直接使用system callls? p. 61, 如此才容易 移植 應用軟體於不同版本的 OS.
2b. 用於系統呼叫之 傳遞多個參數的方法. P. 64, Fig. 2.7, 用一暫存器指示  存參數之記憶體區塊. 
2c. Mac OS X 具有 hybrid 結構,  "hybrid"的含意為何?  那一部分支援POSIX API 函數? hybrid表示兩個不同來源的OS作結合. BSD UNIX的部分.
  
3a. 在擁有兩個processor 的系統上, 有三個processes 同時處於running 狀態是否可能?  為什麼? 可能; 硬體執行緒, p 222.
3b. 呼叫 非同步I/O 之系統呼叫後, process 會被放至那一狀態? ready 狀態
3c.  當一個 process呼叫 fork( )時, OS作什麼動作? p. 116, OS把呼叫 fork( )的process 作複製, 設定其一為父process, 另一為子process
 
4. IPC 機制有: 共享記憶體, 訊息傳送, pipe
4a. send(), mmap() 各用於那一個機制? send()用於訊息傳送, mmap()用於共享記憶體.
4b. IPC 機制, 共享記憶體, pipe 各有何優點? 共享記憶體: 建立後, 存取不透過OS, 所以速度快.  pipe: 可達成資料的同步式存取
  
5a. 產生user-level threads之多執行緒程式, 可獲得之好處? p. 155, 可獲得的好處為 resource sharing, economy
5b. 那些程式組件可被同一process的多個執行緒去分享? p. 162, code, static variable, file
5c. 何謂asynchronous cancellation 與 deferred cancellation? p. 183, 使目標執行緒立刻停止; 目標執行緒定期檢查自身是否應結束.
 
6a. 採non-preemptive排程時, process在何情況會失去CPU使用權? p. 204, (1)當行程呼叫 I/O處理的 system call, (2)當行程執行完畢.
6b. round-robin排程的平均等待時間較長, 然而它的優點是? p. 212, 適合使用於time-sharing OS,  因為 response time 比較短.
6c. 可區分I/O bound行程與CPU bound行程的CPU排程法是? p. 216, multi-level feedback queue 排程法
  
7a. 計算 先佔式 Shortest-Job-First 排程法之平均等待時間. p. 209
7b. 計算 Round-Robin 排程法之平均等待時間, time slice=3. p. 212
  
8a. 當大部分程式都用動態鍊結程式庫來作鍊結, 可有何好處? p. 351, 節省主記憶體空間, 節省執行檔所佔的硬碟空間.
8b. contiguous memory allocation 碰到什麼問題? 可由 paging解決. p. 357, 外部碎片(external fragmentation)之問題.
8c. 64 bit CPU 不採用階層式頁表(page table)的原因為何? p. 374, 頁表需要分成4層以上, 而使主記憶體的存取速度變慢很多.
